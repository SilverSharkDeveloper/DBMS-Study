데이터 베이스 정규화
	관계형 데이터 베이스 모델에서 데이터의 중복성을 제거하여 이상현상을 방지하고 데이터의 일관성을 유지하는 과정

1차 정규화
	테이블 내의 속성값을 원자값으로 구성하는 단계 -> 하나의 컬럼속에는 하나의 값만 존재해야한다. (오류가 발생하진 않지만 유지보수가 매우 힘들다)
								->하나의 컬럼에 여러개의 값이 들어가는경우 테이블을 분리시켜야한다.


2차 정규화
	부분 함수 종속성을 제거하는 단계
	부분 함수 종속 : 릴레이션에서 기본키가 복합키인 경우 기본키를 구성하는 속성 중 일부에게 종속된 경우
		(X,Y) 복합키 식별자 -> Z 종속자 의 관계에서 Y -> Z 또는 X -> Z 의 관계가 성립되는 경우
				

3차 정규화
	이행함수 종속성을 제거하는 단계(각가의 행을 구분하는 컬럼이 아닌 다른 컬럼이 해당 컬럼을 규정하는 상태)
		X식별자 -> Y 종속자 -> Z종속자 의 관계가 성립되는 경우(종속자 Y가 Z를 규정하는 상태)




**함수 종속성 : '데이터베이스의 릴레이션 R(테이블)에서 X와 Y를 R의 부분집합이라고 할 때, X의 값 각각에 대해 Y의 값이 오직 하나로 
결정 될때 Y는 X에 종속 된다고 한다.
정규화가 잘 된 테이블일수록 결정자 X는 PK로 한개이고 종속자가 여러개인 구조를 가진다.
모든 종속자 Y는 기본키 X에 대해서만 종속되며 X를 제외한 다른 속성들이 다른 속성에 종속되지 않는 완전 함수 종속 관계가 정규화가 잘 진행된 테이블이다.


반정규화
	이미 정규화되어있은 것을 이전으로 되돌리는 행위(모든 정규화된 테이블은 조인해서 사용을 해야하므로 속도가 느리다)
	성능향상을 위해서 데이터중복으로 잘못된 값이 들어갈 수 있는 위험을 감수하는 행위

데이터 모델링

개념적 데이터 모델링
	엔티티	 	실체/객체 -> 업무에 필요하고 유용한 정보를 저장하고 관리하기 위한 집합 (테이블로 만들 수 있는데이터들의 집합)
					인스턴스의 집합,속성과 식별자를 포함한다.

논리적 데이터 모델링

물리적 데이터 모델링
		테이블(엔티티)
			행(인스턴스)
			컬럼(속성)
			식별자(PK)
DCL(Data Control Languagae) : 데이터 사용자에 대한 접근권한을 제어하는 언어
	시스템 권한
			1.CREATE SESSION ->데이터베이스 접속 권한
			2.CREATE USER -> 사용자 생성 권한
			3.CREATE TABLE -> 테이블 생성 권한
				...
	오브젝트 권한
		SELECT, INSERT, UPDATE, DELETE
		UPDATE, DELETE 는 WHERE절을 사용하기 때문에 SELECT 권한도 같이 필요하다.

	1.REVOKE 권한 회수
		1시스템 권한 회수 
		REVOKE 시스템권한명 FROM 사용자;
		2.오브젝트 권한 회수
		REVOKE 오브젝트권한명 ON 오브젝트 FROM 사용자;

	2.GRANT 권한 부여
		1.시스템 권한 부여 
			GRANT 시스템권한명 TO 사용자[WITH ADMIN OPTION];
		2.오브젝트 권한 부여
			GRANT 오브젝트 권한,오브젝트권한.. ON 오브젝트 TO 사용자[WITH ADMIN OPTION];
		**WITH ADMIN OPTION 작성시 자신이 받은 권한을 다른 계정에게 줄 수 있음

	부여한 권한을 확인하기 위한 가상 테이블
		SELECT * FROM USER_TAB_PRIVS_MADE;
	부여 받은 권한을 확인하기 위한 가상 테이블
		SELECT * FROM USER_TAB_PRIVS_RECD;
	사용자 계정 생성
		CREATE USER 유저명 IDENTIFIED BY 비밀번호;
