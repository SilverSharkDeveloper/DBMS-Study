GROUP BY 절 ->행의 갯수가 달라짐 -> 요약통계량을 구할 때 보통사용 -> 집계함수와 같이 사용
	같은 컬럼끼리 그룹화를 해준다.
	컬럼이 여러개일 경우 각각의 컬럼이 모두 같은 데이터끼리 그룹화해준다.
	
그룹함수
	GROUP BY 절에 사용할 수 있는 함수

	ROLLUP() : 소계와 총계 정보를 보여준다.
		ROLLUP(컬럼1,컬럼2,...) -> 컬럼1을 기준으로 그룹핑한 결과, 컬럼1의 소계와 총계 정보를 보여준다.
	
	CUBE(컬럼1,컬럼2...) -> 순서 상관없이 써있는 모든 컬럼에 대한 그룹핑 결과와 소계와 총계정보를 보여준다.
	
	GROUPING SETS(컬럼1,컬럼2...) -> 순서 상관없이 써있는 모든 컬럼에 대한 소계만 보여준다.(그룹핑 결과, 총계는 보여주지않는다.)

	**소계,총계를 낼때 기준으로 하는 컬럼이 아닌 컬럼들은 NULL값을 가진다.**
		
	GROUPING(컬럼) ->NULL값을 분류하기위해 보통 사용
		
		-- 해당 컬럼을 모두 묶은 상태라면 결과는1 아니면 0
		--결과가1인지 0인지에 따라서 CASE표현식이나 DECODE()함수를 사용해 분기처리를 할 수 있다.

윈도우 함수
	행과 행간의 관계를 설명하기 위해서 사용하는 함수
	함수 사용결과는 행의 갯수와 일치한다.

	반드시 OVER(PARTITION BY,ORDER BY, WINDOWING) 와 함께 사용해야한다.
	->괄호안에 3개의 절이들어갈 수 있는데 각각이 모두 생략이 가능하다.
	1.PARTITION BY ->윈도우 함수가 적용될 가상의 파티션을 구분, 생략시 전체가 하나의 파티션이 된다.
	2.ORDER BY -> 파티션 내에서 정렬을 할 때 사용, 해당 윈도우함수를 적용할 때 참고하는 정렬이므로 전체 결과에 대한 ORDER BY는 따로 줄 수 있다.
				**전체 결과에대한 ORDER BY절을 사용하지 않을 경우 윈도우함수의 ORDER BY절로 정렬된다.
	3.WINDOWING절 -> 파티션 내에서 어떤 행에 대해서만 적용시킬지 여부 설정
					->항상 기준은 파티션 내에서 현재 보고있는 행을 기준
				ROWS OR RANGE (둘중하나) 시작행 종료행
				ROWS : 물리적인 행을 기준으로
					ROWS BETWEEN 시작행 AND 종료행
						시작행/종료행 
							UNBOUNDED PRECEDING : 파티션 첫번째 행
							값 FOLLOWING : 현재행을 기준으로 값만큼 뒤의 행
							CURRENT ROW : 현재 행	
							값 PRECEDING : 현재행을 기준으로 값만큼 앞의 행
							UNBOUNDED FOLLOWING : 파티션 마지막 행
						**BETWEEN 생략시 시작행부터 CURRENT ROW 까지 이다.

				RANGE : 값을 기준으로 논리적인 범위를 설정 -> 무조건 자신의 값을 기준으로 범위에 해당하는 값까지만 적용을 시킨다.
					 값을 기준으로 하기 때문에 중복값이 있다면 그 중복된 값을 모두 포함
					예를들어 RANGE BETWEEN 1 PRECEDING AND CURRENT ROW 를 했는데 CURRENT ROW 의 값과 다음값이 같다면
							다음 값까지 집계하고 앞의 행의 값도 그전에 행에 중복값이 있다면 그 데이터까지 모두 집계한다.
				


		윈도우 절은 행간의 위치를 기준으로 가져오기 때문에 ORDER BY가 없다면 순서라는 개념자체가 존재하지 않기 때문에 행의 위치를 파악할 수 가 없고
		윈도우절을 사용할 수 없다.
		윈도우절 생략시 기본값은 RANGE BETWEEN UNBOUNDED PRECEDING AND CORRENT ROW 이다.
		

OVER() -> 모든 행을 포함한 집계함수를 구해준다.
OVER(PARTITION BY 컬럼) -> 컬럼별로 파티션을 나누어 집계함수를 구해준다.
OVER(PARTITION BY 컬럼 ORDER BY 컬럼) -> 컬럼별로 파티션을 나누고 정렬한후 파티션 1번행부터 마지막행까지의 집계함수를 구해준다.(WINDOW절 생략시 모든 행)
OVER(PARTITION BY 컬럼 ORDER BY 컬럼 WINDOW절) ->파티션 나누고 정렬후 윈도우절에 맞는 범위로 집계함수를 구해준다.
OVER(ORDER BY 컬럼 WINDOW 절) ->파티션은 모든 행이고 모든 행에 대하여 정렬후 윈도우절에 맞는 범위로 집계함수를 구해준다.
