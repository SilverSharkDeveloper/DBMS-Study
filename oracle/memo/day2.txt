DBMS의 소통 방식

숫자 컬럼이 pk로 사용하는게 유지보수가 편하다.

조합키 
	하나의 pk에 여러 컬럼이 설정된 키
	각 컬럼은 중복이 가능하지만 그룹으로는 중복이 불가능하다.

슈퍼키와 서브키(상속개념 실무용)
	여러 테이블에 공통요소가 있을 때 하나로 묶어줄 테이블을 선언한다.
	이 때 공통요소를 묶은 테이블의 PK를 각 테이블의 FK로 설정함과 동시에 PK로 설정하면
	공통요소를 묶은 테이블의 PK를 슈퍼키라고 하며, 각테이블의 PK를 서브키라고한다.


컴파일 언어 -> 한 파일단위로 컴파일 ->일괄처리 ->효율적 (부분수정시 모두 재해석 ->단점) : 자바
	번역 파일이 생김 -> 느림
스크립트 언어 ->한 줄 단위로 해석 ->개별처리 ->비효율적 (빈번한 수정시 수정된 코드만 재해석 -> 장점) : 
	번역 파일이 없이 바로 실행 -> 빠름

SQL문(쿼리문) -DDL, DML, DCL,TCL
	스크립트 언어, 데이터 베이스와 소통하는 언어

자료형(type) : 용량은 항상 넉넉하게

	
	숫자 NUMBER

	문자열 
		CHAR(n) : 고정형 ->길이가 절대 변하지 않는 값을 넣을 때 사용.
		VARCHAR2(n) : 가변형, 값의 길이만큼 공간이 배정

	날짜 DATE : FROMAT에 맞춰서 날짜를 저장하는 타입
